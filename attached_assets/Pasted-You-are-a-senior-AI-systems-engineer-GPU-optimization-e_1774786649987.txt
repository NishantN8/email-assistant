You are a senior AI systems engineer, GPU optimization expert, and full-stack developer.

Your task is to build a production-grade AI reply system integrated with Gmail API, using a hybrid AI architecture (local GPU + cloud fallback), with automatic model selection to maximize NVIDIA GPU utilization.

---

# 🎯 CORE OBJECTIVE

Build an intelligent reply system that:

* Automatically generates replies for emails
* Uses local GPU (NVIDIA) as primary compute
* Selects the best available local model dynamically
* Falls back to cloud only when necessary
* Learns user tone and improves over time
* Allows user to edit and send replies via Gmail API

---

# 🧠 SYSTEM OVERVIEW

Email → Decision Engine → Reply Engine → UI → Send via Gmail API

---

# ⚙️ STEP 1: LOCAL MODEL DISCOVERY (CRITICAL)

Implement a module that:

* Detects available local models (Ollama or vLLM)
* Benchmarks them based on:

  * latency
  * token/sec
  * VRAM usage
* Selects best model for reply generation

Example:

function selectBestLocalModel(models) {
return models.sort((a, b) =>
b.tokens_per_sec - a.tokens_per_sec
)[0]
}

Supported models:

* llama3 (fast, good)
* mistral (lighter, faster)
* mixtral (better reasoning)
* Llama-3-8B via vLLM (preferred)

---

# ⚡ STEP 2: GPU UTILIZATION

* Ensure CUDA is enabled
* Batch multiple reply requests
* Run inference via vLLM (preferred) or Ollama
* Monitor GPU usage dynamically

generateReply(email, memory, priority)

* If GPU overloaded → cloud
